## HW 1

#### Part 1
1. *Чому аудіо-характеристики винесені в окремий об’єкт `audio_features`, а не зберігаються плоско? Коли таке вкладення вигідне, а коли створює проблеми?*

Винесення в `audio_features` застосовується для логічного групування даних (Data Cohesion).

**Вигідно:** 
- Коли клієнту або фронтенду потрібні всі технічні метрики треку одразу, їх можна отримати одним запитом не перелічуючи десяток окремих полів. 
- Документ стає більш читабельним, відділяючи базові метадані (назва, альбом, тривалість).

**Створює проблеми:** 
- Звернення до вкладених полів вимагає використання dot-notation, що синтаксично трохи складніше.
- Коли потрібно часто оновлювати лише одне поле всередині об'єкта, розробник має бути обережним і обов'язково використовувати оператор $set, інакше можна випадково перезаписати весь об'єкт `audio_features`.

2. *Чому виконавці зберігаються як масив, а не як рядок? Які запити стають простішими?*

- Зберігання у вигляді масиву дозволяє MongoDB створити Multikey Index, де кожен елемент індексується як окреме значення.

- **Чому не рядок:** Пошук артиста у рядку типу "Artist A; Artist B" вимагає використання RegEx для пошуку по підрядку. Це вкрай повільна операція, яка ігнорує звичайні індекси і може давати хибні результати.

**Які запити стають простішими:** 
- Точний пошук: { artists: "Name" } миттєво знайде всі треки виконавця, незалежно від того,скільки їх там.
- Перетини: Використання оператора $all дозволяє легко знайти треки, де працювали конкретні два артисти разом.
- Агрегація: Масив дозволяє використовувати оператор $unwind, щоб "розгорнути" трек і зробити аналітику по кожному артисту окремо (наприклад, порахувати середню популярність виконавця).

3. *Що таке $out і чим він відрізняється від $merge? Коли використовувати кожен?*

Обидва оператори є фінальними стадіями Aggregation Pipeline, які записують результати у колекцію.

- `$out`: Повністю перезаписує цільову колекцію. Якщо колекція вже існувала, вона видаляється, і на її місце записуються нові дані.

    **Використовувати:** міграція даних з нуля, створення повних резервних копій.

- ё$merge`: Оновлює існуючу колекцію. Дозволяє гнучко налаштувати логіку: що робити, якщо документ з таким _id вже існує.

   **Використовувати:** для постійних оновлень, не видаляючи та не перераховуючи мільйони існуючих треків.
